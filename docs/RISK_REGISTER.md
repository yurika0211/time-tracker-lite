# 风险清单

## 1. 高风险

- 计时状态机出错导致时长不准。
- 快照损坏导致历史记录丢失。
- CSV 注入导致表格软件误执行公式。
- 未受控渲染导致页面注入风险。
- 未来同步时静默覆盖数据。

## 2. 中风险

- 大量记录下列表卡顿。
- 迁移脚本漏掉旧数据。
- 依赖升级引入兼容问题。
- 快捷键误触导致用户中断操作。
- 归一化规则过严导致数据无法加载。

## 3. 低风险

- 文案不统一。
- 排版不一致。
- 设置项命名不清。
- 个别边角字段展示不完整。

## 4. 缓解措施

- 先写纯函数测试，再接 UI。
- 任何 schema 变更先写迁移说明。
- 导出前做公式转义。
- 渲染默认走安全文本路径。
- 同步能力默认关闭。
- 恢复前保留上一份快照。

## 5. 风险处理方式

- 高风险：阻止发布。
- 中风险：确认补丁后再发布。
- 低风险：纳入后续迭代。

## 6. 需长期关注的点

- 数据量增长后是否需要分页或虚拟列表。
- 未来同步冲突是否会变复杂。
- 是否需要独立后端或账号体系。
- 是否需要更完整的备份界面。
