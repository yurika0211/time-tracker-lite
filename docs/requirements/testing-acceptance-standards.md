# 测试与验收标准

## 1. 目标

- 验证 P0 闭环是否完整
- 给 P1 / P2 扩展保留统一验收口径
- 把“能跑”变成“可交付”

## 2. 测试分层

### 2.1 单元测试

重点覆盖纯逻辑，不依赖 DOM：

- 时间计算
- 状态机切换
- 标签解析与去重
- 项目去重与归一化
- 统计区间计算
- CSV 行生成
- 数据迁移与归一化

### 2.2 集成测试

重点覆盖模块协作：

- 存储打开、写入、恢复
- 开始 / 暂停 / 继续 / 停止
- 刷新后恢复 active timer
- 搜索、筛选、删除
- 新增项目后回填到选择框
- 清空数据后状态重置
- CSV 导出结果与列表一致

### 2.3 手工验收

重点覆盖真实使用路径：

- 首次进入页面
- 连续开始、暂停、继续、停止
- 刷新恢复
- 新增项目与筛选
- 搜索与删除
- 导出后用表格软件打开
- 异常数据恢复

## 3. P0 验收标准

### 3.1 核心计时

- 页面同时只允许一条 active timer
- 开始后状态切换为进行中
- 暂停后不再累计当前片段
- 继续后从新片段重新累计
- 停止后生成历史记录并清空 active timer
- 刷新后仍能恢复运行态或暂停态
- 运行中修改表单会同步到当前 active timer
- 空格键仅在非输入区域切换计时状态

### 3.2 记录管理

- 历史记录按开始时间倒序展示
- 搜索同时匹配任务名、项目名、标签
- 项目筛选与关键词筛选可以叠加
- 删除只影响目标记录
- 无结果时显示空态
- 过滤条件刷新后保留

### 3.3 项目、客户、标签

- 新增项目时必须去空白、去重
- 常用项目区显示的项目与累计时长一致
- 标签按逗号拆分，去空、去重
- 计费标记在保存后保持一致
- 项目归档后不应默认出现在快捷选择中

### 3.4 导出

- CSV 必须包含任务名称、项目、客户、标签、计费标记、开始时间、结束时间、时长
- 导出结果可直接被表格软件打开
- 行数与当前导出范围一致
- 特殊字符必须正确转义

### 3.5 持久化与恢复

- 刷新页面后数据不丢失
- 清空数据需要二次确认
- 旧数据和缺失字段要能归一化
- 非法或损坏数据不能让页面静默崩溃

## 4. 数据不变量

- `duration_ms >= 0`
- `ended_at >= started_at`
- `active_timer` 只能有一行
- `project`、`client`、`tag` 名称不能为空白
- `time_entry_tags` 不允许重复关联
- 派生统计不落盘

## 5. 验收基线

建议使用以下数据规模做回归验收：

- 1000 条历史记录
- 100 个项目
- 200 个标签
- 1 条运行中的 active timer
- 1 份损坏快照样本
- 1 次刷新恢复
- 1 次导出并用表格软件打开

## 6. 性能门槛

- 开始、暂停、继续、停止应立即反馈
- 搜索和筛选在常规数据量下不应明显卡顿
- 导出期间不能让页面长时间失去响应
- 恢复后首屏应可直接操作

## 7. 失败场景

以下场景必须有明确处理：

- 存储缺失
- schema 不一致
- 时间戳非法
- 标签内容异常
- 项目重名
- localStorage 不可用

## 8. 发布前检查

- 类型检查通过
- 构建通过
- 核心流程人工验收通过
- 刷新恢复通过
- CSV 可打开
- 队列状态已更新
- 文档索引已同步

## 9. 不纳入本次验收

- 登录
- 云同步
- 团队共享
- 自动追踪
- 空闲提醒
- 日历视图
- PDF 导出
