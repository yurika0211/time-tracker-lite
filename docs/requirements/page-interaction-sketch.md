# 页面与交互草案

## 1. 页面结构

### 1.1 顶部区域

- 产品名、当前版本、当前状态
- CSV 导出、清空数据等高风险操作入口
- 顶部区域只放全局动作，不放业务配置

### 1.2 当前计时区域

- 任务名称输入
- 项目选择
- 新增项目入口
- 标签输入
- 计费标记
- 开始、暂停、继续、停止按钮
- 当前计时大字展示
- 当前任务摘要

### 1.3 概览区域

- 今日时长
- 本周时长
- 进行中数量
- 条目总数
- 常用项目列表

### 1.4 记录区域

- 搜索框
- 项目筛选
- 时间条目表格
- 空态展示
- 单条删除

## 2. 页面状态

- **未开始**：没有 active timer，表单可编辑
- **进行中**：计时运行，开始按钮禁用
- **已暂停**：active timer 存在，但暂停累计
- **无记录**：历史列表为空
- **筛选无结果**：有数据但当前筛选后为空
- **数据异常**：读取失败、字段缺失、旧数据归一化失败

## 3. 核心交互

### 3.1 开始 / 暂停 / 继续 / 停止

- 开始后立即生成 active timer
- 运行中任何表单变化都同步到 active timer
- 暂停只停止当前片段累计
- 继续从新片段恢复
- 停止后落为历史记录并清空 active timer

### 3.2 表单联动

- 任务名称、项目、标签、计费标记与当前 active timer 双向同步
- 空值自动归一化，不让脏数据进入事实层
- 新增项目后立即回填到选择框

### 3.3 搜索和筛选

- 关键词搜索同时匹配任务名、项目名、标签
- 项目筛选与关键词筛选可叠加
- 筛选条件在刷新后保留

### 3.4 删除与清空

- 删除只影响单条记录
- 清空数据前必须二次确认
- 高风险动作需要明显提示，不做静默删除

### 3.5 导出

- 导出 CSV 直接下载
- 导出内容默认包含完整历史记录
- 文件名带日期，便于归档

## 4. 后续页面形态

### 4.1 手动补录 / 编辑

- 独立页面或弹窗均可，优先保证字段完整
- 可编辑任务名、项目、标签、计费、开始时间、结束时间
- 时间非法时必须阻止保存

### 4.2 项目与客户管理

- 项目列表支持新增、归档、改名、改色
- 客户列表支持新增、改名、归档
- 客户与项目的绑定关系在此处维护

### 4.3 标签管理

- 展示标签列表与使用次数
- 支持重命名、合并、删除
- 删除标签前需要确认其影响范围

### 4.4 报表页面

- 今日、周、月、自定义范围
- 按项目、客户、标签汇总
- 支持导出汇总结果

### 4.5 设置页面

- 默认项目、默认计费、快捷键说明
- 数据备份和恢复入口
- 未来同步开关和账号入口预留

## 5. 键盘和无障碍

- 空格：开始、暂停、继续切换
- Enter：提交当前表单
- Esc：关闭弹窗或取消当前编辑
- 表单元素必须有明确标签
- 按钮状态变化要可被屏幕阅读器感知

## 6. 响应式规则

- 桌面优先，主流程保证一屏可见
- 中等宽度下改成上下堆叠
- 小屏下表格横向滚动，不压缩字段到不可读
- 高风险操作在窄屏下仍保留确认步骤

## 7. 交互约束

- 所有即时保存动作都要有失败回退路径
- 自动同步只改当前活跃实体，不改历史记录
- 不允许通过界面制造重复项目名和非法时间区间
- 重要状态变化要立即刷新界面

## 8. 与实现对齐

- 当前单页原型先满足核心计时闭环
- 后续再拆分为独立管理页和报表页
- 任何页面新增字段，都要同步更新数据结构和导出字段
